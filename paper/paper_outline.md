### §1.3 Focused Challenge (1 paragraph)

> **Core challenge**: "How to integrate heterogeneous knowledge sources — structured commonsense (ConceptNet), unstructured crawled data (internet slang), and LLM-synthesized domain knowledge (service scenarios) — into a unified retrieval-augmented base that a standard BERT model can use for instruction semantic understanding in service scenarios."

This decomposes into three sub-problems:
1. **Unified representation**: Three sources differ in format, granularity, and noise
2. **Semantic drift suppression**: Pure vector retrieval generates false positives
3. **Validation on constrained model**: Proving the gain is from knowledge, not from a larger model

### §3.3 Data Cleaning and Quality Control

**§3.3.1 Cleaning Pipeline for Crawled Data**
- DeepSeek V4-Flash normalization: remove redundant suffixes, assign 4-class emotion labels, flag service relevance
- Local filter: regex-based removal of NBA/anime/gaming/exam/political domains
- Filter rate: ~16% (723 out of 4,441)

**§3.3.2 Multi-Agent Scoring for LLM-Generated Data**
- Generator: V4-Pro assigns polarity scores (-5 to +5) on both positive and negative dimensions
- Auditor: V4-Flash checks three red-line rules: polarity conflict, logic inversion, insufficient intensity
- Pipeline: 5,000 entries → 4,987 Approved (99.7%)
- 13 entries flagged Manual_Review (auditor format errors, not content)

### §3.5 Vectorization and Dual-Path Retrieval

**§3.5.1 Sentence-BERT Encoding**
- Model: shibing624/text2vec-base-chinese (768-dim)
- FAISS IndexFlatL2 for exact search
- Backend comparison: FlatL2 (100% recall) vs IVFFlat (97.2%) vs IVFPQ (93.5%) vs HNSW (98.1%) vs Annoy (92.8%)

**§3.5.2 Dual-Path Hybrid Retrieval**

**Path 1: Exact Substring Matching**
- Scan KB for keyword appearing as substring in utterance
- Longest match selected when multiple hits
- Precision: 0.92
- Coverage: ~45% of test utterances

**Path 2: FAISS Vector Retrieval with Cosine Threshold**
- Encode utterance → top-k neighbors (k=3) → cosine validation
- Candidates with cos < τ=0.72 discarded
- τ determined by grid search on dev set (sweep τ ∈ [0.50, 0.90], F1 maximum at τ=0.72)

**Results**:

| Strategy | Precision | Recall | F1 |
|------|:--:|:--:|:--:|
| Exact only | 0.92 | 0.45 | 0.60 |
| Vector only (τ=0.72) | 0.78 | 0.76 | 0.77 |
| **Dual-path** | **0.85** | **0.82** | **0.83** |

### §6.2 Limitations

| # | Limitation | How to Address |
|:--:|------|------|
| L1 | Class imbalance in KB (sarcasm-heavy, anger-underrepresented) | ClassBalancer mechanism ongoing; stated as future work |
| L2 | Input-level augmentation (string concatenation treats knowledge same as text) | Dual-encoder architecture as future direction |
| L3 | Single-dataset evaluation (LLM-generated only) | Cross-distribution validation with real user data needed |
| L4 | Pure text modality (no speech prosody, physical context) | Multimodal extension planned |

### §6.3 Future Work (Thesis Scope)

1. **Chain-of-Thought reasoning**: Replace BERT with DeepSeek V4 for multi-step inference
2. **Conflict quantification**: Dual-stream metric C = |Literal_Score − State_Score| / 10
3. **Closed-loop response**: Task suspension, apology playback, human escalation when C exceeds threshold
